# 1. 한눈에 보기 (Executive Summary)

## 1.1 제품 개요

| 항목          | 내용                                                                |
| ------------- | ------------------------------------------------------------------- |
| **제품명**    | Kidsroad (키즈로드)                                                 |
| **슬로건**    | "아이와 함께하는 모든 순간, 키즈로드가 안내합니다"                  |
| **제품 유형** | 모바일 퍼스트 웹 서비스 (향후 앱 확장)                              |
| **핵심 가치** | 아이 연령과 부모 조건에 맞는 축제·행사 정보를 한 곳에서 빠르게 제공 |

## 1.2 핵심 가치 제안 (Value Proposition)

> **"흩어진 정보를 모으고, 부모 관점으로 정제하여, 검색 시간을 줄여드립니다"**

**Before Kidsroad (현재)**

- 네이버 검색 → 블로그 → 지도 → 공식 사이트 → 다시 후기 확인
- 평균 30분 이상 소요, 정보 신뢰도 불확실

**After Kidsroad (목표)**

- 한 번의 검색으로 필요한 모든 정보 확인
- 5분 이내 의사결정 완료

## 1.3 MVP 핵심 기능 요약

1. **연령별 맞춤 필터** - 0-2세 / 3-5세 / 6-9세 / 10세+
2. **부모 체크리스트 필터** - 유모차, 주차, 실내, 무료 여부
3. **통합 행사 데이터** - 공공 API + 크롤링으로 전국 행사 수집
4. **정제된 상세 페이지** - 부모가 필요한 핵심 정보만 제공
5. **상황별 큐레이션** - 이번 주말, 비 오는 날 등 맞춤 추천

# 2. 배경 및 문제 정의 (Why)

## 2.1 시장 상황

### 시장 규모

- 국내 육아·키즈 산업: **약 40조 원** (2024년 기준)
- 가족 체험·놀거리 시장: 매년 **15% 이상 성장**
- 주말 가족 외출 검색량: 월평균 **500만 건 이상**

### 시장의 특징

- "아이와 갈 곳" 관련 검색량은 높지만, **정리된 플랫폼이 없음**
- 정보는 많으나 **산재되어 있고 품질이 일정하지 않음**
- 아이 동반 가족은 광고 단가가 높아 **수익화에 유리**

## 2.2 사용자 페인포인트

### 현재 부모들의 정보 탐색 여정

```
[1단계] 네이버에서 "아이와 갈만한 곳" 검색
    ↓
[2단계] 블로그 3-5개 클릭하며 후기 확인
    ↓
[3단계] 네이버 지도에서 위치 확인
    ↓
[4단계] 공식 홈페이지에서 운영시간/비용 확인
    ↓
[5단계] 다시 블로그로 돌아가 추가 후기 확인
    ↓
[결과] 30분 이상 소요, 그래도 불확실함
```

### 핵심 페인포인트 3가지

| 페인포인트              | 상세 설명                                          | 영향              |
| ----------------------- | -------------------------------------------------- | ----------------- |
| **정보 분산**           | 검색, 지도, 블로그, 공식 사이트를 오가며 정보 수집 | 시간 낭비, 피로감 |
| **아이 맞춤 정보 부재** | "3살 아이도 괜찮을까?" 판단 어려움                 | 불안한 의사결정   |
| **부모 필수 정보 누락** | 유모차, 주차, 수유실 등 현장 정보 확인 어려움      | 현장 도착 후 당황 |

### 사용자 인용 (실제 맘카페 의견)

> "주말마다 검색하는데 30분은 기본이에요. 그래도 막상 가면 유모차 못 끌고 가는 경우도 있고..."

> "3살인데 괜찮을까? 블로그 보면 다 '좋다'만 써있어서 판단이 안 돼요"

> "정보가 너무 흩어져 있어요. 한 곳에서 다 볼 수 있으면 좋겠어요"

## 2.3 기회 영역

### 경쟁 환경 분석

| 서비스                 | 유형        | 강점                 | 한계                 | 위협도 |
| ---------------------- | ----------- | -------------------- | -------------------- | ------ |
| **네이버 검색/블로그** | 검색 포털   | 사용자 多, 정보량 多 | 정보 산재, 광고성 글 | 낮음   |
| **네이버 지도**        | 지도 서비스 | 위치 기반 접근       | 행사/축제 정보 약함  | 낮음   |
| **여기어때/야놀자**    | 예약 플랫폼 | 결제 시스템 강함     | 키즈 필터 미흡       | 중간   |
| **키즈카페 앱들**      | 버티컬 앱   | 특정 영역 집중       | 축제/행사 정보 없음  | 낮음   |
| **지자체 사이트**      | 공공 서비스 | 공식 정보            | UI 열악, 모바일 ✕    | 낮음   |

### 시장 기회 (Market Gap)

> **"정보는 넘쳐나지만, 부모 관점으로 정제된 단일 플랫폼은 없다"**

**Kidsroad가 노리는 포지션:**

- 데이터: 공공 + 민간 통합
- 필터: 아이 연령 + 부모 조건 특화
- UX: 모바일 최적화 + 빠른 의사결정 지원

# 3. 제품 정의 (What)

## 3.1 제품 비전

### 단기 비전 (1년)

> "아이와 갈 곳을 찾을 때 가장 먼저 떠오르는 서비스"

### 장기 비전 (3년)

> "아이와 함께하는 모든 활동을 연결하는 패밀리 라이프스타일 플랫폼"

## 3.2 핵심 차별점

| 차별 요소           | Kidsroad                          | 경쟁사 현황      |
| ------------------- | --------------------------------- | ---------------- |
| **연령별 필터**     | 0-2 / 3-5 / 6-9 / 10세+           | 없거나 매우 단순 |
| **부모 체크리스트** | 유모차, 주차, 수유실, 위험요소 등 | 거의 없음        |
| **통합 데이터**     | 공공API + 크롤링 + 사용자 제보    | 분산되어 있음    |
| **상황별 큐레이션** | 날씨, 주말, 무료 등 맞춤 추천     | 없음             |
| **후기 품질**       | 짧고 핵심적, 아이 반응 중심       | 광고성 글 多     |

## 3.3 타겟 사용자

### Primary 페르소나: "바쁜 워킹맘 지현씨"

| 항목     | 내용                                                |
| -------- | --------------------------------------------------- |
| **이름** | 김지현 (가명)                                       |
| **나이** | 35세                                                |
| **직업** | IT 회사 마케터                                      |
| **가족** | 남편, 4세 딸                                        |
| **특징** | 평일 바쁨, 주말에 아이와 시간 보내고 싶음           |
| **니즈** | 빠른 검색, 신뢰할 수 있는 정보, 아이 맞춤 추천      |
| **불만** | "검색에 30분 쓰고도 막상 가면 별로인 경우가 많아요" |

### Secondary 페르소나: "꼼꼼한 전업맘 수진씨"

| 항목     | 내용                                               |
| -------- | -------------------------------------------------- |
| **이름** | 박수진 (가명)                                      |
| **나이** | 32세                                               |
| **직업** | 전업주부                                           |
| **가족** | 남편, 2세 아들, 5세 딸                             |
| **특징** | 평일에도 아이들과 외출 잦음, 정보 수집에 시간 투자 |
| **니즈** | 무료 행사, 연령별 적합성, 실내/실외 구분           |
| **불만** | "둘 다 데리고 갈 수 있는 곳 찾기가 너무 어려워요"  |

## 3.4 사용자 시나리오

### 시나리오 1: 주말 나들이 계획

**상황:** 금요일 저녁, 지현씨가 주말에 아이와 갈 곳을 찾고 있음

**As-Is (현재)**

1. 네이버에서 "서울 아이 축제" 검색
2. 블로그 5개 읽음 (15분)
3. 괜찮아 보이는 곳 지도에서 확인 (5분)
4. 공식 사이트에서 정보 확인 (5분)
5. 유모차 되는지 다시 블로그 검색 (5분)

→ **총 30분, 그래도 확신 없음**

**To-Be (Kidsroad 사용 시)**

1. Kidsroad 접속
2. 필터 설정: 서울 / 3-5세 / 실외 / 주차 가능
3. 추천 리스트 확인
4. 상세 페이지에서 필요한 정보 한눈에 확인

→ **총 5분, 확신을 가지고 결정**

### 시나리오 2: 비 오는 날 급하게 찾기

**상황:** 토요일 아침, 비가 와서 야외 계획 취소. 급하게 실내 갈 곳 필요

**As-Is:** "비 오는 날 아이랑" 검색 → 키즈카페만 나옴 → 다른 옵션 찾기 어려움

**To-Be:**

1. 필터: 오늘 가능 / 실내 / 내 위치 근처
2. 체험전, 박물관, 키즈카페 등 다양한 옵션 확인
3. 비 오는 날 추천 큐레이션 참고

→ **3분 만에 결정**

### 시나리오 3: 무료 행사 찾기

**상황:** 월급 전, 무료로 아이와 갈 수 있는 곳 찾기

**As-Is:** 무료 행사 정보가 흩어져 있어 찾기 어려움, 지자체 사이트 UI가 불편해서 포기

**To-Be:**

1. 필터: 무료 / 이번 주
2. 공공데이터 기반 무료 행사 리스트 확인
3. 연령대 맞는지 태그로 바로 확인

→ **2분 만에 결정**

# 4. 기능 상세 (How)

## 4.1 기능 구조도 (Feature Map)

```
Kidsroad
├── 🏠 메인 홈
│   ├── 검색바
│   ├── 빠른 필터 (지역, 연령, 실내/실외)
│   ├── 이번 주 추천 큐레이션
│   └── 인기 행사 리스트
│
├── 🔍 검색/탐색
│   ├── 키워드 검색
│   ├── 필터 조합 검색
│   │   ├── 지역 필터
│   │   ├── 연령 필터
│   │   ├── 부모 체크리스트 필터
│   │   └── 기간 필터
│   └── 검색 결과 리스트
│
├── 📄 행사 상세
│   ├── 기본 정보 (제목, 기간, 장소)
│   ├── 부모 필수 정보 (체크리스트)
│   ├── 지도 연동
│   ├── 공식 링크
│   └── 관련 행사 추천
│
├── ❤️ 저장/기록
│   ├── 찜한 행사
│   └── 최근 본 행사
│
└── 📝 큐레이션
    ├── 이번 주말 추천
    ├── 날씨별 추천
    └── 무료 행사 모음
```

## 4.2 핵심 기능 상세 설명

### 기능 1: 연령별 맞춤 필터

**목적:** 내 아이 나이에 맞는 행사만 빠르게 찾기

**연령 구분 기준:**

| 연령대 | 대상        | 특징                                     |
| ------ | ----------- | ---------------------------------------- |
| 0-2세  | 영아        | 유모차 필수, 짧은 활동 시간, 수유실 필요 |
| 3-5세  | 유아        | 체험 활동 가능, 집중 시간 30분-1시간     |
| 6-9세  | 초등 저학년 | 교육적 콘텐츠 선호, 긴 활동 가능         |
| 10세+  | 초등 고학년 | 독립적 활동 가능, 스릴/모험 선호         |

**동작 방식:**

- 복수 선택 가능 (예: 0-2세 + 3-5세 동시 선택)
- 선택 시 즉시 리스트 갱신
- 연령 태그가 행사 카드에 표시됨

### 기능 2: 부모 체크리스트 필터

**목적:** 현장에서 당황하지 않도록 사전 조건 확인

**체크리스트 항목:**

| 항목          | 설명                | 아이콘  |
| ------------- | ------------------- | ------- |
| 실내/실외     | 날씨 영향 여부 판단 | 🏠 / 🌳 |
| 유모차 가능   | 유모차 접근성       | 👶      |
| 주차 가능     | 자차 이용 가능 여부 | 🚗      |
| 무료          | 입장료 무료 여부    | 💰      |
| 수유실        | 영아 동반 시 필수   | 🍼      |
| 기저귀 교환대 | 영아 동반 시 필수   | 👶      |

**데이터 확보 방식:**

- 공공데이터 API에서 제공하는 경우 자동 매핑
- 미제공 시 Rule-based 추정 (예: "실내"가 제목에 포함 → 실내로 분류)
- 사용자 제보로 보완 (Phase 2)

### 기능 3: 행사 카드 리스트

**목적:** 스캔하기 쉬운 형태로 행사 정보 제공

**카드 구성 요소:**

- 썸네일 이미지
- 태그 (연령, 실내/실외, 무료 등)
- 행사명
- 장소
- 기간
- 찜 버튼

### 기능 4: 행사 상세 페이지

**목적:** 의사결정에 필요한 모든 정보를 한 페이지에서 제공

**페이지 구성:**

| 섹션                | 포함 정보                               |
| ------------------- | --------------------------------------- |
| **헤더**            | 제목, 대표 이미지, 찜 버튼              |
| **핵심 정보**       | 기간, 장소, 비용, 운영시간              |
| **부모 체크리스트** | 유모차, 주차, 수유실 등 아이콘으로 표시 |
| **추천 연령**       | 해당 행사에 적합한 연령대               |
| **상세 설명**       | 행사 소개 텍스트                        |
| **위치**            | 지도 (네이버/카카오 지도 연동)          |
| **외부 링크**       | 공식 홈페이지, 예약 페이지              |
| **관련 행사**       | 비슷한 조건의 다른 행사 추천            |

### 기능 5: 상황별 큐레이션

**목적:** 검색 없이도 상황에 맞는 추천 제공

**큐레이션 유형:**

| 유형      | 예시                             | 갱신 주기        |
| --------- | -------------------------------- | ---------------- |
| 시간 기반 | "이번 주말 추천", "이번 달 축제" | 주 1회           |
| 날씨 기반 | "비 오는 날 실내 추천"           | 실시간 (Phase 2) |
| 비용 기반 | "무료로 즐기는 행사 TOP 5"       | 주 1회           |
| 지역 기반 | "서울 근교 당일치기"             | 월 1회           |

## 4.3 기능 우선순위

### Phase 1: MVP (필수)

| 우선순위 | 기능                 | 필요 이유               |
| -------- | -------------------- | ----------------------- |
| P0       | 공공데이터 API 연동  | 데이터 없이 서비스 불가 |

## 8.1 용어 정의 (Glossary)

| **MAU**             | Monthly Active Users, 월간 활성 사용자 수                   |
| **DAU**             | Daily Active Users, 일간 활성 사용자 수                     |
