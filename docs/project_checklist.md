1인 개발 프로젝트로 수익화까지 고려할 때 개발 전/중에 결정하고 준비해야 할 사항들을 정리한 체크리스트입니다.

---

## 1. 사업적 결정 사항

- [ ] **사업자 등록**: 수익화 시점에 필요 (개인/법인, 간이/일반과세자)
- [ ] **서비스 이용약관**: 런칭 전 필수 작성
- [ ] **개인정보처리방침**: 런칭 전 필수 작성
- [ ] **결제 시스템**: PG사 선택 (토스페이먼츠, 아임포트 등)
- [ ] **환불 정책**: 미리 정해두지 않으면 분쟁 소지

---

## 2. 기술 인프라 결정

| 항목             | 상태      | 선택              | 비고                                      |
| ---------------- | --------- | ----------------- | ----------------------------------------- |
| 도메인           | ⬜ 미정   |                   | 가비아, Cloudflare 등                     |
| 호스팅/클라우드  | ✅ 결정됨 | Vercel + Supabase | Next.js 배포는 Vercel, DB/인증은 Supabase |
| CI/CD 파이프라인 | ✅ 결정됨 | Vercel 자동 배포  | GitHub 연동 시 자동                       |
| 모니터링         | ⬜ 미정   |                   | Sentry, Supabase Dashboard 활용 가능      |
| 로깅 전략        | ⬜ 미정   |                   | 어디까지 남길지, 보관 기간                |
| 파일 스토리지    | ✅ 결정됨 | Supabase Storage  | 이미지 등 파일 저장                       |

## 3. 보안 관련

| 항목                     | 상태      | 선택            | 비고                                   |
| ------------------------ | --------- | --------------- | -------------------------------------- |
| 인증/인가 방식           | ✅ 결정됨 | Supabase Auth   | OAuth2 소셜 로그인 지원                |
| API 보안 (Rate Limiting) | ⬜ 미정   |                 | Supabase에서 일부 제공, 추가 설정 필요 |
| CORS 설정                | ⬜ 미정   |                 | 배포 시 설정 필요                      |
| 민감정보 관리            | ✅ 결정됨 | 환경변수 (.env) | Vercel 환경변수 사용                   |
| SSL 인증서               | ✅ 결정됨 | 자동 (Vercel)   | Vercel 배포 시 자동 HTTPS              |
